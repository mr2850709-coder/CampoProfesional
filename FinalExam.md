### 2. GIT, VERSIONAMIENTO Y TRABAJO PROFESIONAL
4. Explica con tus palabras el flujo básico de trabajo en Git (commit, push, pull, branch, merge).

Commit: guardo cambios  con una descripción de lo que hice.

Push: envío mis commits al repositorio, GitHub.

Pull: descargo los cambios que otros han hecho para mantener mi copia actualizada.

Branch: creo una rama para trabajar en una característica sin afectar el código principal.

Merge: une los cambios de una rama con otra

5. ¿Por qué es importante usar control de versiones en proyectos reales?

Es importante porque permite tener historial de cambios, recuperar versiones anteriores si algo falla y organizar el desarrollo de nuevas funciones sin afectar la versión estable del proyecto.

6. ¿Cómo aplicarías GitHub en un proyecto profesional o universitario en equipo?

Lo aplicaría para organizar el proyecto creando ramas por tareas, asignando issues y asegurando que todos trabajemos con la última versión del código.
